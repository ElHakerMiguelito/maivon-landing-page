Guía de personalización

Todo el contenido de la página vive en un solo objeto CONFIG. Para adaptar la plantilla a cualquier negocio, solo editas ese objeto — sin tocar HTML ni CSS.

?

¿Cómo funciona?

💡 El archivo MAIVON Landing Page.html tiene un objeto CONFIG en el <script> al final. El JavaScript lee ese objeto y construye toda la página automáticamente: textos, colores, servicios, horarios, links de WhatsApp y más.

La arquitectura tiene tres capas:

  • HTML — solo estructura vacía (divs con IDs). No tiene texto directo.
  • CSS — estilos con variables :root que se sobreescriben desde CONFIG.colores.
  • CONFIG — único punto de verdad. Todo el contenido está aquí.
→

Pasos rápidos para un negocio nuevo

1

Abre el archivo HTML

Busca el bloque const CONFIG = { — está cerca del final del archivo, dentro del <script>.

2

Edita la sección Identidad

Cambia nombre, giro, tagline y whatsapp. Esto actualiza el título, navbar, hero y footer.

3

Ajusta los colores

En CONFIG.colores cambia verde por el color de acento del negocio. El resto de la página se actualiza automáticamente.

4

Sube los logos

Coloca los archivos de logo en la carpeta uploads/ y actualiza las rutas en CONFIG.logo.

5

Llena servicios, testimonios y ubicación

Edita cada sección con los datos reales del negocio. Los arrays se pueden extender o reducir sin límite.

1

Identidad del negocio

Datos base que se usan en múltiples secciones — navbar, hero, footer, meta SEO.

nombre Nombre comercial del negocio. Aparece en el navbar (si no hay logo), en el footer y en el título del browser. nombre: "Mi Negocio"
giro Categoría o tipo de negocio. Aparece debajo del nombre en el footer y en la meta descripción. giro: "Restaurante de comida japonesa"
tagline Frase corta de marca. Se usa internamente como referencia; el texto visible del hero se controla en hero.titulo. tagline: "Sabor que enamora"
descripcion Descripción larga para SEO. Aparece en la meta etiqueta del <head> — no se muestra en pantalla. descripcion: "Restaurante japonés en el centro de la ciudad. Sushi fresco, ramen y más."
contacto.tipo
contacto.valor Define el canal principal de contacto del negocio. Todos los botones de la página usan este valor. Ver la sección Método de contacto para opciones completas. contacto: { tipo: "whatsapp", valor: "526671234567" }
2

Colores

🎨 Al cambiar cualquier valor en CONFIG.colores, el JS lo aplica como variable CSS en :root y toda la página se recolorea instantáneamente.
CampoUso en la páginaEjemplo
verde Color de acento principal. Botones, íconos, precios, barra de carga, links hover, textos destacados. "#0CAA58"
verdeOscuro Estado hover de los botones de acento. Debe ser más oscuro que verde. "#09844a"
oscuro Texto principal, fondos oscuros (hero, testimonios, footer), hamburger. "#1A1A1D"

fondoAlt Fondo de secciones alternas (Por qué nosotros). Debe ser ligeramente más oscuro que fondo. "#EAEAE4"

3

Logos

Sube los archivos a la carpeta uploads/ y apunta las rutas aquí. Si un logo no existe, deja el valor en "".

logo.horizontal Navbar (arriba a la izquierda). Logo con texto en una línea. Si está vacío, se muestra el nombre en texto. horizontal: "uploads/logo-horizontal.png"
logo.vertical Logo con texto apilado (icono arriba, nombre abajo). Reservado para usos futuros. vertical: "uploads/logo-vertical.png"
logo.icono Solo el símbolo/ícono, sin texto. Se usa en el splash screen de carga. icono: "uploads/icono.png"
💡 Usa archivos PNG con fondo transparente para que el logo se vea bien sobre cualquier fondo. Para el splash screen, el logo icono funciona mejor.
4

Navegación

nav.ctaTexto Texto del botón verde en el navbar (desktop). ctaTexto: "WhatsApp"
nav.ctaMensaje Mensaje preescrito para el botón del navbar. Se usa como texto en WhatsApp o como asunto en email. Se ignora si el tipo de contacto es Instagram o teléfono. ctaMensaje: "Hola, me gustaría recibir información sobre sus servicios."
nav.links Array de links del menú. Cada uno tiene etiqueta (texto visible) y href (ancla o URL). Para añadir/quitar links, agrega o borra objetos del array. links: [ { etiqueta: "Servicios", href: "#servicios" }, { etiqueta: "Contacto", href: "#ubicacion" }, ]
5

Hero (sección de inicio)

Primera pantalla que ve el visitante. Fondo oscuro con dos botones de llamada a la acción.

hero.eyebrow Texto pequeño en verde sobre el título. Generalmente el giro del negocio. eyebrow: "Taquería desde 1985"
hero.titulo Título grande del hero. Acepta HTML — usa <span class='accent'>texto</span> para colorear una palabra en verde. titulo: "El mejor taco <span class='accent'>de la ciudad</span>"
hero.subtitulo Párrafo descriptivo debajo del título. 1-2 oraciones recomendadas. subtitulo: "Sabores auténticos preparados con ingredientes frescos cada día."
hero.ctaTexto
hero.ctaMensaje Texto e mensaje del botón principal de contacto. El ctaMensaje se usa en WhatsApp/email; se ignora en Instagram y teléfono. ctaTexto: "Hacer un pedido", ctaMensaje: "Hola, quiero hacer un pedido."
hero.btn2Texto
hero.btn2Href Texto y destino del botón secundario (borde blanco). Generalmente apunta a la sección de servicios o menú. btn2Texto: "Ver menú", btn2Href: "#servicios"
6

Servicios / Productos

Muestra tarjetas con nombre, descripción, precio y botón de contacto. El canal (WhatsApp, Instagram, email…) se hereda de CONFIG.contacto. El grid se ajusta automáticamente al número de items.

servicios.items Array de servicios. Cada uno tiene: nombre, descripcion, precio y mensaje (texto que acompaña el contacto — usado en WhatsApp y email). { nombre: "Plan Básico", descripcion: "Descripción breve del servicio o producto.", precio: "$500/mes", // deja "" si no quieres mostrar precio mensaje: "Hola, me interesa el Plan Básico.", // para WA / email }
💡 Puedes tener tantos items como quieras. Con 3 items se ven 3 columnas en desktop. Con 2 ó 4+ el grid se adapta. Si el negocio no tiene precios, deja precio: "" y no aparecerá.
7

Por qué nosotros

4 propuestas de valor con ícono, título y descripción corta.

porqueNosotros.items Cada item tiene titulo, texto e icono. El icono es un nombre de la tabla de íconos (ver sección Íconos). { titulo: "Experiencia real", texto: "15 años en el mercado nos respaldan.", icono: "bolt", }
8

Galería

Grid de imágenes de proyectos o del negocio. Actualmente muestra placeholders — se reemplazan subiendo imágenes.

⚠️ Para usar imágenes reales: sube los archivos a uploads/, luego agrega la ruta en cada item: { alt: "descripción", src: "uploads/foto1.jpg" }. El JS usará <img> en lugar del placeholder cuando exista src.
galeria.items Array de imágenes. Cada item tiene alt (descripción accesible) y opcionalmente src (ruta de la imagen). { alt: "Local del restaurante" }, // sin src → placeholder { alt: "Platillo estrella", src: "uploads/platillo.jpg" }
9

Testimonios

testimonios.items Array de reseñas. Cada una tiene texto, nombre, negocio (o cargo) e iniciales (2 letras para el avatar). { texto: "Excelente servicio. Lo recomiendo totalmente.", nombre: "María G.", negocio: "Boutique Flores", iniciales:"MG", }

Ubicación y horarios

ubicacion.direccion Dirección completa que aparece en el bloque de info. direccion: "Calle Hidalgo 45, Col. Centro, Culiacán, Sin."
ubicacion.mapaQuery Texto que se pasa a Google Maps para el embed y el link "Ver en Google Maps". Puede ser la misma dirección o el nombre del lugar. mapaQuery: "Calle Hidalgo 45, Col. Centro, Culiacán"
ubicacion.horarios Array de rangos de días con su horario. Cada fila tiene dia y hora. { dia: "Lunes – Viernes", hora: "9:00 am – 7:00 pm" }, { dia: "Sábado", hora: "10:00 am – 3:00 pm" }, { dia: "Domingo", hora: "Cerrado" },

CTA Final

Sección de cierre con fondo verde y un botón de contacto. El canal se hereda de CONFIG.contacto.

ctaFinal.titulo
ctaFinal.subtitulo Título y párrafo de la sección. Deben ser directos y orientados a la acción. titulo: "¿Listo para ordenar?", subtitulo: "Escríbenos y te atendemos en minutos.",
ctaFinal.btnTexto
ctaFinal.mensaje Texto del botón y mensaje de contacto. El campo mensaje se usa en WhatsApp/email; se ignora en Instagram y teléfono. btnTexto: "Hacer mi pedido", mensaje: "Hola, quiero hacer un pedido ahora.",

Footer

footer.redesSociales Array de redes. Cada una tiene red, url, usuario e icono (nombre del ícono disponible: instagram, facebook, linkedin, twitter). { red: "Instagram", url: "https://instagram.com/mi_negocio", usuario: "@mi_negocio", icono: "instagram", }
footer.copyright Usa {año} para el año actual y {nombre} para el nombre del negocio. Se rellenan automáticamente. copyright: "© {año} {nombre}. Todos los derechos reservados."

Método de contacto

Todos los botones de la página (navbar, hero, servicios, CTA final) usan el canal definido en CONFIG.contacto. Cambiar tipo y valor actualiza toda la página automáticamente — ícono, link y comportamiento.

💡 Solo hay dos campos: tipo (el canal) y valor (el dato de ese canal). Nada más. 
// ── WhatsApp (por defecto)
contacto: { tipo: "whatsapp",  valor: "526674683970" },
//  valor = número con código de país, sin signos
//  México +52 → "52667..."  ·  USA +1 → "1555..."
//  Los campos 'mensaje' de cada botón se envían como texto pre-escrito.

// ── Instagram
contacto: { tipo: "instagram", valor: "mi_negocio" },
//  valor = usuario sin @ · abre instagram.com/mi_negocio/
//  Los campos 'mensaje' se ignoran (Instagram no los soporta).

// ── Email
contacto: { tipo: "email",     valor: "hola@minegocio.com" },
//  valor = dirección de correo
//  Los campos 'mensaje' se usan como asunto del email.

// ── Teléfono
contacto: { tipo: "telefono",  valor: "+526671234567" },
//  valor = número con + y código de país · abre la app de llamadas
//  Los campos 'mensaje' se ignoran.
